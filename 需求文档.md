# 钓鱼社交社区H5应用 - 需求文档

## 一、产品定位

### 核心价值
为钓鱼爱好者提供的内容分享与互动社区

### 目标用户
钓鱼爱好者（休闲钓友、专业钓手、新手入门）

### 核心体验
- 快速浏览优质钓鱼内容
- 便捷分享钓鱼生活
- 发现同好钓友互动

---

## 二、功能模块详解

### 模块1：首页内容流

#### 双Tab切换
| Tab | 说明 | 展示内容 |
|-----|------|----------|
| 关注 | 用户主动关注的钓友动态 | 仅显示关注用户的动态，按时间倒序 |
| 推荐 | 智能推荐+最新内容 | 混合推荐算法，下方显示 |

#### 推荐算法逻辑（按优先级）
1. **关注优先**：关注用户的动态优先展示
2. **互动热度**：基于点赞、评论、收藏、分享数据加权
3. **兴趣推荐**：根据用户浏览/点赞历史推荐相似内容
4. **时间衰减**：新发布内容有额外权重

#### 内容卡片展示
```
┌─────────────────────────────────┐
│ [头像] [昵称]          [⋮ 更多]  │
│ 钓获分享 · 3分钟前              │
│ ┌─────────────────────────────┐ │
│ │  [图片1] [图片2] [图片3]     │ │
│ │  [图片4] [图片5]            │ │
│ └─────────────────────────────┘ │
│ 今天运气不错，去XX水库钓了半天   │
│ 拉上来5条鲤鱼，最大的这条有4斤   │
│ 📍 XX水库（点击跳转钓点详情）    │
│ ────────────────────────────  │
│ ❤️ 128  💬 32  ⭐ 56  📤 分享   │
└─────────────────────────────────┘
```

#### 内容状态
- ✅ 正常可见：审核通过且未删除
- ⏳ 审核中：新发布内容，等待审核（系统审核）
- ⚠️ 审核未通过：内容违规，通知用户

#### 交互操作
- 点击头像 → 进入用户主页
- 点击图片 → 大图预览，左右滑动
- 点击钓点 → 跳转钓点详情（关联地图）
- 点击评论 → 弹出评论列表
- 双击内容 → 快速点赞动画
- 点击⋯ 更多 → 展开操作菜单

#### 更多操作菜单
```
┌─────────────────┐
│ 举报          │
│ 不感兴趣      │
│ 屏蔽此内容    │
│ 分享          │
└─────────────────┘
```

#### 加载机制
- 首次加载：拉取10条
- 下拉刷新：重新加载最新内容
- 上拉加载：每次加10条，到底提示"没有更多了"

### 模块2：发布动态

#### 发布入口
- 底部导航 + 按钮
- 首页顶部快捷入口（用户引导）

#### 发布流程
```
步骤1：选择内容类型
  ├─ 📷 拍照
  ├─ 🖼️ 从相册选择
  ├─ 🎬 录视频
  └─ ✍️ 纯文字

步骤2：编辑内容
  ├─ 图片/视频编辑（裁剪、排序）
  ├─ 填写文字描述
  ├─ 选择分类标签
  └─ 关联钓点（选填）

步骤3：预览确认
  ├─ 预览发布效果
  ├─ 返回修改
  └─ 确认发布
```

#### 图片编辑功能
- 裁剪：自由裁剪、1:1、3:4、4:3、16:9比例
- 排序：拖拽调整图片顺序
- 最多支持9张图片

#### 文字输入限制
- 最多500字
- 实时显示字数统计
- 支持@提及钓友（输入@触发搜索）

#### 分类标签（单选）
| 分类 | 说明 |
|------|------|
| 晒鱼获 | 展示钓鱼成果，分享喜悦 |
| 钓点推荐 | 推荐好钓点，分享经验 |
| 钓鱼技巧 | 分享钓鱼技巧、心得 |
| 装备交流 | 讨论钓具、饵料、装备 |

#### 关联钓点（选填）
- 点击从地图选择
- 搜索钓点名称
- 或使用当前位置
- 关联后自动显示钓点名称和地址

#### 草稿机制
- 编辑过程中自动保存草稿
- 进入发布页检测是否有未完成草稿
- 提示"是否继续编辑草稿"
- 草稿箱管理

#### 提交后状态
- 显示"提交审核中..."
- 自动跳转到"我的动态"页
- 审核通过后显示"发布成功"，可分享
- 审核失败显示原因，可修改重新提交

#### 编辑规则
- 发布后不能编辑
- 如需修改，删除后重新发布

#### 评论规则
- 展示按时间倒序
- 回复以缩进形式平铺展示（不嵌套）
- 支持无限层级的回复展示
- 点击"更多"收起长回复
- 单条评论最多100字

- 点赞评论
- 回复评论（自动填充"回复@用户名"）
- 举报评论
- 删除自己的评论

#### 回复通知
- 回复他人时，被回复人收到通知
- 回复自己的评论时不通知
- 多人回复同一条评论时，每人只通知一次

### 模块8：举报系统

#### 举报类型
| 举报对象 | 举报理由选项 |
|----------|--------------|
| 动态 | 色情低俗、政治敏感、虚假信息、广告垃圾、人身攻击、其他 |
| 用户 | 虚假身份、恶意刷屏、人身攻击、其他 |
| 评论 | 恶意灌水、人身攻击、其他 |

#### 举报流程
1. 点击"举报"
2. 选择举报理由
3. 填写补充说明（选填）
4. 提交举报
5. 显示"举报已提交，感谢反馈"

#### 举报处理
- 举报后进入审核队列
- 系统自动审核（敏感词、图片识别）
- 人工审核复核
- 处理结果不通知举报人（保护隐私）

- 动态卡片右上角"分享"按钮

## 四、数据模型

### 动态数据
```javascript
{
  id: String,              // 动态ID
  userId: String,          // 发布者ID
```

## 五、用户权限体系

### 用户状态
| 状态 | 说明 | 权限 |
|------|------|------|
| 未登录 | 游客状态 | 可浏览，不可互动 |
| 已注册 | 普通用户 | 可浏览、发布、互动 |
| 审核中 | 新注册等待审核 | 可浏览，不可发布 |
| 已封禁 | 违规用户 | 提示账号已被封禁 |

### 操作权限
| 操作 | 游客 | 普通用户 | 已封禁 |
|------|------|----------|--------|
| 浏览动态 | ✅ | ✅ | ❌ |
| 发布动态 | ❌ | ✅ | ❌ |
| 点赞评论 | ❌ | ✅ | ❌ |
| 关注用户 | ❌ | ✅ | ❌ |

## 六、内容审核机制

### 系统审核（自动）
- 敏感词过滤
- 图片AI识别（色情、暴力、政治）
- 违规自动拦截

### 审核结果
- ✅ 通过：立即发布
- ❌ 不通过：通知用户具体原因

### 人工审核（抽查）
- 随机抽查新用户发布内容
- 处理举报内容
- 封禁恶意用户

### 审核时效
- 系统审核：实时（<1秒）

## 十一、核心业务流程

### 登录流程
```
1. 用户输入手机号
2. 点击"获取验证码"
3. 用户输入验证码
4. 提交登录请求
5. 后端验证验证码
6. 返回用户token
7. 前端存储token
8. 跳转到首页
```

### 发布动态流程
```
1. 选择内容类型（图片/视频/文字）
2. 编辑内容（裁剪、排序、描述）
3. 选择分类标签
4. 关联钓点（选填）
5. 预览确认
6. 提交发布
7. 系统自动审核
8. 审核通过→发布成功
9. 审核失败→提示原因
```

### 评论互动流程
```
1. 点击动态评论
2. 弹出评论列表
3. 输入评论内容
4. 提交评论
5. 更新评论数
6. 如果回复他人→发送通知给被回复人
7. 刷新评论列表
```

### 关注用户流程
```
1. 点击用户头像/推荐钓友
2. 进入用户主页
3. 点击"关注"按钮
4. 发送关注请求
5. 更新关注数和粉丝数
6. 按钮状态变为"已关注"
7. 对方收到新粉丝通知
```
